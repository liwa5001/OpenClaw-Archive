## 🎯 核心决策记录

### 2026-03-08 21:40 - 饮食数据系统

**决策内容：**
```
原设计：问进食窗口（用户自己估算）
新设计：问三餐内容 → 系统计算卡路里

示例问题：
"今天三餐吃了什么？请详细描述：
- 早餐：____:____，内容：________________
- 午餐：____:____，内容：________________
- 晚餐：____:____，内容：________________"

系统处理：
1. 解析食物内容
2. 查询食物数据库
3. 计算总卡路里
4. 记录进食时间
5. 分析 16+8 执行情况
```

**食物数据库来源：**
- 中国食物成分表（标准版）
- USDA FoodData Central
- 常见中餐菜品数据库

**未来扩展：**
- 宏量营养素分析（碳水/蛋白质/脂肪）
- 微量元素追踪（维生素、矿物质）
- 饮食质量评分（基于营养密度）

## 📊 数据系统设计原则

### 统一标准（全六堡适用）

1. **数据格式**
   - Markdown + YAML Front Matter
   - 日期格式：YYYY-MM-DD
   - 数值带单位

2. **存储结构**
   ```
   data/
   └── {domain}/
       ├── {metric}-tracker.md  # 长期追踪
       └── raw/                  # 原始数据
   ```

3. **趋势分析**
   - 每周生成趋势图
   - 月度汇总报告
   - 跨领域关联分析

4. **可视化**
   - Markdown 表格
   - ASCII 图表（简单场景）
   - 外部工具导出（复杂图表）
